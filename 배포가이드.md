# 간편결제 뉴스 대시보드 — GitHub Pages 배포 가이드

## 파일 구조

```
payment-news-dashboard/          ← 저장소 이름 (자유롭게 설정)
├── index.html                   ← 대시보드 웹페이지
├── data/
│   └── news.json                ← 뉴스 누적 데이터 (자동 업데이트)
├── scripts/
│   └── collect_news.py          ← 뉴스 수집 스크립트
└── .github/
    └── workflows/
        └── collect-news.yml     ← 자동 실행 스케줄 (매일 오전 8시 KST)
```

---

## 1단계 — GitHub 저장소 생성

1. [https://github.com/new](https://github.com/new) 접속
2. Repository name: `payment-news-dashboard` (원하는 이름 가능)
3. **Public** 선택 (GitHub Pages 무료 사용 조건)
4. **Create repository** 클릭

---

## 2단계 — 파일 업로드

방법 A (권장) — Git 사용:
```bash
git clone https://github.com/[내_계정]/payment-news-dashboard
# 위 파일들을 모두 복사한 뒤
git add .
git commit -m "초기 설정"
git push origin main
```

방법 B — GitHub 웹 업로드:
- 저장소 페이지에서 **Add file → Upload files**
- 파일을 드래그 앤 드롭하여 업로드 (폴더 구조 유지 필요)

---

## 3단계 — 네이버 API 키 등록 (Secrets)

1. 저장소 → **Settings → Secrets and variables → Actions**
2. **New repository secret** 클릭, 두 개 등록:

| Name | Value |
|------|-------|
| `NAVER_CLIENT_ID` | 네이버 개발자센터 Client ID |
| `NAVER_CLIENT_SECRET` | 네이버 개발자센터 Client Secret |

> 네이버 API 키 발급: https://developers.naver.com/apps/#/register
> → 애플리케이션 등록 → 검색 API 선택

---

## 4단계 — GitHub Pages 활성화

1. 저장소 → **Settings → Pages**
2. Source: **Deploy from a branch**
3. Branch: `main` / `/ (root)`
4. **Save** 클릭
5. 약 1~2분 후 `https://[내_계정].github.io/payment-news-dashboard/` 접속 가능

---

## 5단계 — 첫 번째 수동 실행

Actions 탭 → **간편결제 뉴스 수집** → **Run workflow** 클릭

뉴스 수집 완료 후 `data/news.json`이 업데이트되고 웹사이트에 기사가 표시됩니다.

---

## 이후 자동 운영

- 매일 오전 8시(KST) 자동으로 뉴스 수집 및 JSON 업데이트
- GitHub Actions → Commits에서 실행 이력 확인 가능
- 90일 이전 데이터는 자동 삭제

---

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 뉴스가 표시되지 않음 | news.json 비어있음 | Actions에서 수동 실행 |
| Actions 실패 | API 키 오류 | Secrets 값 재확인 |
| 페이지 404 | Pages 미활성화 | Settings → Pages 재확인 |
